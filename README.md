# ReFurnish — вертикальный прототип (MVP core)

## О проекте

**ReFurnish** — это маркетплейс для апсайклинга мебели и обновления интерьера.  
Платформа соединяет владельцев мебели, которые хотят обновить вещи или интерьер, с мастерами по ремонту, реставрации и апсайклингу.

Основная ценность сервиса:
- пользователь не выбрасывает мебель, а даёт ей «вторую жизнь»;
- мастера получают заказы на свои услуги;
- со временем вокруг платформы формируется комьюнити и база успешных кейсов по апсайклингу.

В данном репозитории реализуется **вертикальный прототип** — узкий, но сквозной срез функциональности: от интерфейса пользователя до базы данных, строго в рамках нижеперечисленных функций.

---

## Функциональность вертикального прототипа

Вертикальный прототип ReFurnish покрывает базовый сценарий:  
**«Клиент находит мастера и публикует проект на апсайклинг мебели, мастер видит этот проект и может откликнуться».**

### 1. Пользовательские роли (в прототипе)

В вертикальном прототипе поддерживаются два типа пользователей:

- **Клиент** — владелец мебели или жилья, который хочет обновить интерьер.
- **Мастер** — исполнитель, предлагающий услуги ремонта, реставрации или апсайклинга.

Администратор, партнёры и другие роли **в прототип не входят** (их функции будут добавлены позже).

---

### 2. Для клиента (владельца мебели)

В рамках вертикального прототипа клиент может:

1. **Зарегистрироваться и авторизоваться**
   - Простая регистрация по email и паролю.
   - Вход в систему и выход из аккаунта.

2. **Просматривать список мастеров**
   - Список мастеров с краткой карточкой (имя/никнейм, специализация, базовое описание).
   - Базовые фильтры:
     - по городу / региону (геолокация в упрощённом виде — выбор города из списка);
     - по специализации (например: «перетяжка мягкой мебели», «реставрация дерева», «покраска»).
   - Сортировка по рейтингу (рейтинг — пока статическое поле / заглушка, без полноценной системы отзывов).

3. **Открывать карточку мастера**
   - Расширенное описание профиля мастера;
   - Примеры работ (в прототипе — изображения/скриншоты, загружаемые мастером или предзаполненные);
   - Основные специализации и ориентировочная стоимость.

4. **Создавать проект по апсайклингу**
   - Форма создания проекта:
     - название проекта;
     - текстовое описание задачи;
     - выбор типа мебели (например: стул, стол, диван и т.п.);
     - загрузка 1–3 фотографий (упрощённая загрузка);
     - примерный бюджет (число);
     - желаемый срок выполнения (дата или текстовое поле).
   - Сохранение проекта в статусе «Опубликован».

5. **Просматривать свои проекты**
   - Список созданных проектов с краткой информацией (название, статус, бюджет, мастер/отклики).
   - Просмотр детальной карточки проекта.

---

### 3. Для мастера

В рамках вертикального прототипа мастер может:

1. **Зарегистрироваться как мастер**
   - Создание профиля мастера (отдельный тип аккаунта):
     - имя/никнейм;
     - краткое описание;
     - специализации (несколько тегов);
     - город;
     - ориентировочная стоимость работ (формат «от X»).

2. **Просматривать список открытых проектов**
   - Список проектов клиентов, находящихся в статусе «Открыт для откликов».
   - Базовые фильтры:
     - по городу;
     - по типу мебели.

3. **Откликаться на проект (упрощённый отклик)**
   - Возможность оставить отклик:
     - краткий комментарий;
     - предлагаемая стоимость;
     - предлагаемая дата начала.
   - Отклики пока видит только клиент в карточке проекта (без полноценного чата и сложной логики назначения).

4. **Просматривать свои отклики**
   - Список проектов, на которые мастер откликался.

---

### 4. Что **НЕ** входит в вертикальный прототип

Следующие возможности описаны в общей спецификации ReFurnish, но **не реализуются** в рамках этого вертикального прототипа:

- Полноценная **система бонусов RePoints** (начисление, обмен, партнёрские предложения).
- **Видео-гайды** и интеграция с YouTube/Rutube.
- **Продвинутый чат** (в прототипе возможен только упрощённый отклик без real-time общения).
- Система **отзывов и рейтингов** с формированием рейтинга по результатам заказов.
- **Оплата и биллинг** (интеграция с ЮKassa / Stripe).
- **Панель администратора**: модерация контента, управление партнёрками и акциями.
- Отчёты и аналитика (отчёты по активности, доходам мастеров и т.п.).
- Мобильное приложение (в прототипе фокус на веб-версии, даже если фронтенд пишется с учётом будущего RN).

## Технологический стек (для вертикального прототипа)

Согласно спецификации проекта:

- **Backend**: Go (основной сервис API), PostgreSQL как основная база данных.
- **Frontend (web)**: React / SPA, адаптивный интерфейс под desktop
